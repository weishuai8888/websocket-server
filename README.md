步骤 B：部署流程
  你有三种主要方式部署：

  方式一：代码仓库自动部署 (推荐)
   1. 将你的代码（包含 Dockerfile）提交到 GitHub、GitLab 或 Gitee。 
   2. 登录 微信云托管控制台 (https://cloud.weixin.qq.com/cloudrun)。
   3. 新建服务 -> 选择“关联代码仓库”。
   4. 选择你的仓库和分支。
   5. 关键配置：
       * Dockerfile 路径：填 Dockerfile (默认)。
       * 容器监听端口：填 8080 (必须与你代码中的端口一致)。
   6. 点击创建/部署。

  方式二：本地代码上传
   1. 在控制台新建服务 -> 选择“本地代码上传”。
   2. 将当前文件夹压缩为 .zip 包。
   3. 上传压缩包。
   4. 配置端口为 8080 并部署。

  方式三：CLI 命令行部署
  如果你安装了微信云托管 CLI，可以使用命令行部署。
   1. 登录 CLI。
   2. 运行 wxcloud run deploy (具体命令取决于 CLI 版本)。

  注意事项
   * WebSocket 支持：微信云托管支持 WebSocket，但要注意连接超时设置（默认可能有超时限制，需在业务逻辑中处理心跳包，或者在云托管配置中调整）。
   * 域名：部署成功后，你会获得一个公网访问的默认域名，但正式使用通常需要绑定备案过的自定义域名。

第四步：配置服务参数

  在云托管控制台设置：

  - 服务名称：例如 snake-game-ws
  - 端口号：8080（需与代码中一致）
  - CPU/内存：0.25核/0.5G（起步配置）
  - 实例数量：最小1个，最大按需扩容
  - 环境变量：数据库连接等配置

第五步：获取服务域名

  部署成功后，云托管会自动分配域名：
  - 格式：wss://snake-game-ws-[环境ID].sh.run.tcloudbase.com
  - 已自带 SSL 证书，支持 wss 协议
  - 无需额外配置域名和证书
